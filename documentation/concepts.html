Concept

This application is built using three primary components:

Database

A MySQL database serves as the application’s data store, with enhanced durability through the use of a persistent volume and persistent volume claim to ensure data is stored permanently and securely.

Backend

A Flask application operates as the backend, managing data exchange between the database and frontend. This component acts as the bridge, providing the communication between the data layer and user interface.

Frontend

The frontend is developed with HTML and React, and is served by an NGINX server. This component presents the dog adoption features, displays available dogs and provides a form for submitting new adoption requests.

Overview

Each component is encapsulated in its own Docker container, with dedicated Dockerfiles, Kubernetes deployments and services. Additional concepts

Additional concepts implemented in this project, not depicted in the image, include storing sensitive information in secrets.csv, using DNS to resolve the host address of the database service, and configuring horizontal auto-scaling for the Flask backend via the flask_hpa file.
