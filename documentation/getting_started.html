Getting Started

Prerequisites

This project uses Kubernetes and Docker as core technologies for application deployment and management. Before running the application, please ensure that both Docker and Kubernetes are installed on your system.

To install Docker, Docker Buildx (for building Docker images), and Kubernetes, you can use the following commands

sudo apt install docker.io
sudo apt install docker-buildx
sudo apt-mark hold kubelet kubeadm kubectl

In addition, we utilize Kubernetes command-line tools such as kubectl and kubeadm for managing clusters. Kubectl is especially critical as it provides the necessary commands for running and interacting with the application on Kubernetes. Further installation guidance is available in the course description on GitHub.

Application Setup

To start, Docker images must be created for each application component and added to the Kubernetes folder. Ensure that the image tag (version) specified during the build process matches the tag referenced in the Flask app deployment configuration. Also do not forget to build all of the services, deployments and other crucial Kubernetes components.

  1. Navigate to the directory containing the Dockerfile for the application component.
  2. Build and save the Docker image with the following commands: 
    sudo docker buildx build --output type=docker --tag <app name>:<version>  .
docker save <app_name>:<version> -o <app_name>.tar
  3. Import the Docker image to Kubernetes:
    4. sudo ctr -n=k8s.io images import <app_name>.tar

To deploy Kubernetes

  1. Navigate to the directory containing Kubernetes deployment files.
  2. Apply all deployment and service configurations with:
    3. kubectl apply -f .
