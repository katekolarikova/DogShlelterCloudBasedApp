Tests

This project includes both backend (Flask) and frontend (React) unit tests to ensure functionality and reliability across the API and UI. The following sections outline how to run these tests and provide a brief overview of what the tests cover.

Flask API Unit Tests

The Flask API tests validate the backend endpoints to ensure proper data retrieval, insertion, and error handling.

Running Tests

  1. In the flask_app directory run: 
    python -m unittest test_app.py

Tests overview

  1. /health endpoint:Verifies that the health check endpoint returns a 200 OK status and a healthy status message.
  2. /api/dogs endpoint:Tests that dog data is fetched and formatted correctly, and validates error handling for failed database connections.
  3. /api/adoption endpoint: Checks successful form submission and database insertion for new dog entries, along with error handling for failed submissions.

Flask API Unit Tests

The React tests validate frontend components using the @testing-library/react package, covering UI rendering, user interaction, and error handling within the application’s core components.

Running Tests

  1. In the react_frontend directory run: 
    npm test

Tests overview

  1. App Component Tests:
    • Located in App.test.js
    • Tests main navigation rendering and default page content.
  2. DogCard Component Tests:
    • Located in DogCard.test.js
    • Validates correct display of dog details: name, breed, age, and description.
  3. Form Component Tests:
    • Located in Form.test.js and FormInvalidInput.test.js
    • Form.test.js: Covers successful form submission and error handling on submission failure.
    • FormInvalidInput.test.js: Handles validation checks for required fields, and displays error messages for invalid email and phone formats.
  4. Homepage Component Tests:
    • Located in Homepage.test.js
    • Checks data fetching, rendering of dog details, and error handling for data load failures.
